# How to Set Smarter Notifications in Google Calendar

*Originally published on [Schedule Calendar Blog](https://calendar-extension.site/blog/google-calendar-notifications.html)*

---

Google Calendar ships with a default ten-minute pop-up notification for every event. This setting makes sense in the abstract — it gives you a reminder before each commitment — and creates significant noise in practice, where it fires for every standup, every lunch block, every personal reminder you set three months ago.

Notification fatigue is the predictable result. When every alert carries the same urgency signal, the signal stops being meaningful. You start dismissing all alerts reflexively, including the ones that matter. The notification that should have reminded you to prepare for an important client call gets swept away with the same gesture as the reminder to take a water break.

Fixing this requires about five minutes and a clearer philosophy about what notifications are for.

Notifications are for events that would genuinely benefit from a specific reminder — not for everything on the calendar. Events you attend every week without fail do not need reminders. Events you scheduled yourself and are aware of usually do not either. Notifications add value for external commitments with preparation requirements, for events that fall at unusual times, and for anything with consequences if missed.

The practical approach: change the global default to fifteen minutes, which gives most people more useful lead time than ten. Then disable notifications for recurring internal meetings. Add a day-before email reminder for external client meetings. Use the pop-up notification only for events that genuinely need a real-time alert.

At the calendar level, you can configure different defaults for different calendars. A work calendar might use pop-up notifications; a personal calendar might use email-only or no notifications at all.
