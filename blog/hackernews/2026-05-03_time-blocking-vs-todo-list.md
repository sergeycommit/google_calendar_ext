# Time Blocking vs. To-Do Lists: They Solve Different Problems

*Hacker News post · May 3, 2026*

---

The debate framing of "time blocking vs. to-do lists" is a false dichotomy. They answer different questions. Using only one is like using only a spec or only a calendar — you're missing half the system.

## What each tool actually does

**To-do list:** A capture and prioritization tool. Holds everything that needs to happen. Low friction to add. Easy to scan for "what's outstanding." Does not answer: when will this happen, how long will it take relative to available time, or whether there's realistic capacity to do it this week.

**Calendar blocks:** A scheduling and commitment tool. Places work in time, makes it visible alongside other commitments, forces explicit trade-offs between items when capacity is limited. Does not answer: what are all the things that need to get done, or how to prioritize when everything feels urgent.

## The combined system

The list is infinite. The calendar is finite. That tension is the productive constraint.

Weekly: review the list, select the tasks that *must* happen this week, block time for them. This is where the calendar reveals the lie of the list — when you try to put everything on the calendar, you discover there isn't time for it. The calendar forces you to pick.

Daily: within a work block, use the list as the queue. The calendar defines the *when*; the list defines the *what* within that time.

End of day: sync. Which tasks moved? Do they need a new block or can they wait for the next weekly review?

## The failure modes

**List-only:** You know everything you need to do and never have time to do it. The gap between list and reality is permanent and stressful.

**Calendar-only:** Over-scheduled rigidity. Nothing captures the work that doesn't have a specific time yet. Ad-hoc tasks either get lost or displace blocks.

**Both, combined:** The list captures; the calendar schedules. Neither is authoritative alone.
